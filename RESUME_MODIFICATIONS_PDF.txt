🏗️ ARCHITECTURE DU CODE
════════════════════════════════════════════════════════════════════════════

1. CONSTANTES DE CONFIGURATION (lignes 11-21)
   ├─ COLORS: Palette de couleurs
   ├─ PAGE_MARGIN: Marge de page (40px)
   └─ LINE_HEIGHT: Hauteur de ligne standard

2. FONCTION addHeader() (lignes 23-42)
   ├─ Crée l'en-tête professionnel
   ├─ Affiche titre, sous-titre, dates
   ├─ Retourne la position Y pour le contenu
   └─ Fond gris clair

3. FONCTION addFooter() (lignes 44-56)
   ├─ Ajoute pied de page
   ├─ Ligne de séparation
   ├─ Branding et numéro de page
   └─ Appelée une fois par page

4. FONCTION drawTable() (lignes 58-117)
   ├─ Crée un tableau structuré
   ├─ Gère en-têtes avec bordures
   ├─ Lignes alternées blanc/gris
   ├─ Alertes visuelles (couleur d'alerte)
   ├─ Pagination automatique
   └─ Retourne position finale et numéro de page

5. FONCTION generatePDF() (lignes 119-400)
   ├─ Fonction principale de génération
   ├─ Type INVENTAIRE:
   │  ├─ Crée tableau 4 colonnes
   │  ├─ Affiche statuts avec alertes
   │  └─ Ajoute résumé
   ├─ Type BILAN:
   │  ├─ Crée 4 cartes statistiques
   │  ├─ Ajoute tableau résumé
   │  └─ Format moderne avec couleurs
   └─ Gère pagination et footers

🚀 UTILISATION
════════════════════════════════════════════════════════════════════════════

GÉNÉRATION AUTOMATIQUE:
1. Clic sur bouton "📄 PDF" dans l'écran Rapports
2. Frontend appelle GET /api/rapports/:id/pdf
3. Backend recharge les données fraîches
4. Fonction generatePDF() créé le PDF
5. PDF binaire envoyé au frontend
6. Téléchargement automatique

PARAMÈTRES PASSÉS:
├─ rapport: Objet Rapport MongoDB complet
│  ├─ _id: Identifiant unique
│  ├─ type: 'inventaire' ou 'bilan'
│  ├─ dategeneration: Date création
│  └─ generePar: User object (avec .nom)
│
└─ data: Données fraîches recalculées
   ├─ Pour inventaire: {produits, lots, totalLots, totalProduits}
   └─ Pour bilan: {totalProduits, totalLots, totalEmplacements, quantiteTotal}
