⚡ DÉMARRAGE EN 5 MINUTES
════════════════════════════════════════════════════════════════════════════

ÉTAPE 1: Vérifier les dépendances (30 secondes)
────────────────────────────────────────────────
✓ PDFKit est déjà installé (dans package.json)
✓ Rien d'autre à installer

ÉTAPE 2: Redémarrer le backend (1 minute)
────────────────────────────────────────────────
cd backend
npm start

Vérifier les logs:
✓ "Server running on port 5000"
✓ Pas d'erreur

ÉTAPE 3: Frontend lancé (1 minute)
────────────────────────────────────────────────
cd frontend
npm start

Vérifier:
✓ Application se charge
✓ Vous pouvez vous connecter

ÉTAPE 4: Générer un rapport (1 minute)
────────────────────────────────────────────────
1. Aller à "Rapports"
2. Cliquer "📋 Inventaire" ou "📊 Bilan"
3. Attendre "✅ Succès"
4. Rapport apparaît dans la liste

ÉTAPE 5: Télécharger le PDF (1 minute)
────────────────────────────────────────────────
1. Cliquer "📄 PDF" sur un rapport
2. PDF se télécharge
3. Ouvrir avec lecteur PDF
4. ✅ Résultat magnifique!

🆘 DÉPANNAGE
════════════════════════════════════════════════════════════════════════════

PROBLÈME: PDF ne se télécharge pas
SOLUTION:
  □ Vérifier que backend est lancé
  □ Vérifier logs backend pour erreurs
  □ Vérifier droits utilisateur (admin/gestionnaire)

PROBLÈME: PDFs vides
SOLUTION:
  □ Vérifier qu'il y a des données dans BD
  □ Vérifier logs console pour erreurs

PROBLÈME: Tableau mal aligné
SOLUTION:
  □ Normalement pas de problème avec logique améliorée
  □ Vérifier version PDFKit: npm list pdfkit

PROBLÈME: Couleurs incorrectes
SOLUTION:
  □ Vérifier lecteur PDF supporte couleurs
  □ Essayer lecteur différent (Acrobat, navigateur)

PROBLÈME: Erreur 500
SOLUTION:
  □ Vérifier logs backend
  □ Vérifier rapport existe dans BD
  □ Vérifier données complètes
